# Slovník

| Termín | Význam ve veřejném profilu v0.2.2 |
|---|---|
| **Autorita (authority)** | Zdroj, který smí určit, co v projektu platí. |
| **Decision owner** | Člověk s konečnou odpovědností za rozhodnutí. |
| **Kontrakt (contract)** | Dohoda o odpovědnosti, hranicích, vstupech a výstupech. |
| **Scope** | Přesný rozsah změny včetně zákazů a hranic. |
| **Owner** | Osoba odpovědná za dokument nebo rozhodnutí. |
| **DRAFT** | Návrh k review; není závaznou instrukcí. |
| **ACTIVE** | Dokument schválený pro aktuální použití v dané oblasti. |
| **SUPERSEDED** | Dokument nahrazený novější verzí. |
| **ARCHIVED** | Historický dokument, který není aktuální autorita. |
| **Document conflict** | Rozpor mezi dokumenty; musí být vyřešen před prací. |
| **Project state** | Stav host projektu, oddělený od životního cyklu dokumentu. |
| **Module** | Doménová oblast, kterou host projekt skutečně vybral. |
| **Change surface** | Konkrétní druh změny s jedním vlastnickým modulem. |
| **Bootstrap** | Přizpůsobení kontraktů pravdě konkrétního host projektu. |
| **READY** | Odvozený stav konzistentního bootstrapu, nikoli oprávnění k provedení. |
| **BLOCKED** | Výsledek kontroly, která nesmí bezpečně pokračovat. |
| **Evidence** | Důkazní stopa spojující intent, review, autorizaci a výsledek. |
| **Reference-only** | Profil pro čtení, validaci a adaptaci, ne kompletní runtime. |
| **Integrity check** | Hashová kontrola deklarovaného obsahu veřejného balíčku. |

Anglické názvy zůstávají v kontraktech záměrně. Při rozporu má přednost
konkrétní autoritativní dokument a jeho lidský owner před tímto slovníkem.
