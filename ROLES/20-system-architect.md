# Role: System Architect

## Purpose

The System Architect is the human decision owner for one adopted host project.

## Responsibilities

- formalize or accept bounded intent;
- approve, reject, or narrow scope;
- decide selected modules and acceptance boundaries;
- authorize progression after architectural review.

## Limits

This role does not become an AI-executed role. Decisions must remain explicit
in host-project records rather than inferred from a code change.
