# Quickstart

Use this kit as a starting point for a local governance layer in an existing
repository.

1. Read [the fictional example](./examples/fictional-project/) to see the
   intended level of detail.
2. Name one human decision owner for the repository change.
3. Copy the templates into a project-local documentation area.
4. Record the repository purpose, boundaries, architecture rules, codebase
   conventions, and approved integration points.
5. Select only the templates that describe a real project need.
6. Review proposed AI-assisted work against those recorded boundaries before
   implementation begins.

This kit supplies a reference structure. Each adopting project remains
responsible for its own implementation, tooling, access controls, testing,
and release process.
